# 安全政策

## 支持版本

当前只维护主分支上的最新版本。

## 报告安全问题

如果你发现以下问题，请不要在公开 issue 中贴出敏感细节：

- API Key、Token、Cookie 或其他凭据泄露风险。
- 个人信息、简历内容或聊天内容泄露风险。
- 权限误用、跨站数据读取或未预期的网络请求。
- 可以导致扩展执行未授权操作的问题。

请先创建一个不包含敏感细节的 GitHub issue，说明问题类型和影响范围。维护者会在 issue 中确认后续沟通方式。

## 公开日志注意事项

提交 bug 前请先脱敏日志和截图。不要公开：

- 候选人姓名、手机号、邮箱、微信号。
- 简历原文、聊天记录、公司内部信息。
- Cookie、Token、Authorization、API Key。
- 自定义 AI 服务地址中不可公开的路径或参数。
